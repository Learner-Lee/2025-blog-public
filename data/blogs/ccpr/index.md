# Prompt: 帮我系统学习 Ralph Loop（给 Claude Code 的任务书）

## T — Task（任务）
你是我的**技术导师 + 陪练**。我要学习 "Ralph Loop"（Geoffrey Huntley 提出的 AI 编码循环工作流模式）。请为我产出一份**实践导向的学习文档**，并在我准备好之后，**带我一步步动手跑通**。

交付物分两阶段：
1. **阶段一（文档）**：生成一份 `ralph-loop-learning.md`，包含三层模块（基础必学 / 进阶 / Master）。
2. **阶段二（陪练）**：文档写完后，暂停并问我："要从哪个模块开始动手？" 之后按我选择的模块，带我在本机真实跑一遍。

## R — Request（具体要求）

### 文档结构（严格遵守）
```
# Ralph Loop 学习路线
## 0. 它是什么 & 为什么存在（100 字以内白话解释）
## 1. 基础必学模块
## 2. 进阶模块
## 3. Master 模块
## 4. 什么时候【不该】用 Ralph Loop（重要）
## 5. 动手路线图（下一步做什么）
```

### 每个知识点必须包含这 4 块（缺一不可）

- **🔧 这是什么**：一句话说清楚。
- **✅ 加上它/这么做，你会得到**：具体、可观察的效果。例：「循环会在任务完成后自动退出，而不是烧 token 到你睡醒」。
- **⚠️ 改动它，会发生什么**：列举 2-3 个常见修改和后果。例：「把 stop 条件删掉 → 无限循环，钱包警告」。
- **🧪 最小可跑示例**：3-15 行的代码/命令，**必须能直接复制运行**，不要伪代码。

### 风格要求
- 新手友好，避免黑话；必须用黑话时先解释。
- 多用「如果你…就会…」句式，少用「理论上…」。
- 每个模块开头写一句"**学完这节你能做到什么**"。
- 不要长篇大论，一个知识点控制在 1 屏以内。

## A — Action（执行步骤）

按顺序执行，**不要跳步**：

1. **先搜资料**：用 WebSearch 搜 "Ralph Loop Geoffrey Huntley"，找到原始博客文章并读完。如果搜不到一手资料，**停下来告诉我**，不要自己编。
2. **列大纲**：把三层模块的知识点清单先发给我看，等我说 "OK 继续" 再展开写。
3. **写文档**：按上面的结构和四块格式，生成 `ralph-loop-learning.md`。
4. **自检**：写完后逐条对照 Request 里的要求打勾，缺了就补。
5. **暂停提问**：问我从哪个模块开始动手。
6. **陪练**：根据我选的模块，创建一个 `ralph-practice/` 目录，带我真实跑一遍。每一步先解释再执行，执行后让我确认现象是否符合预期。

## C — Context（上下文）

- 我是**新手**，刚接触 Claude Code，Shell 基础会一点（cd / ls / cat 级别）。
- 我的目标：能自己写 `PROMPT.md` + 循环脚本，让 AI 自动迭代完成一个中型任务。
- 重要前提：**Ralph Loop 不是 Claude Code 的内置功能**，它是一种"外层 shell 循环 + Claude Code CLI 命令"的组合用法。文档里要讲清楚这个边界，不要让我以为它是某个按钮。
- 我担心的事：token 烧太快、循环停不下来、AI 跑偏。文档里请针对这三点给出预防措施。
- 环境：macOS / Linux（bash 或 zsh）。

## E — Example（样板，请照此风格写每个知识点）

> ### 1.2 PROMPT.md —— 循环的"方向盘"
> **🔧 这是什么**  
> 一个纯文本文件，里面写着你想让 AI 每轮循环都做的事。循环脚本每次都把它喂给 Claude Code。
>
> **✅ 加上它，你会得到**  
> - AI 每一轮都有一致的目标，不会"忘记自己在干嘛"。
> - 你可以在循环跑着的时候直接编辑它，下一轮就生效 —— 相当于"热更新"指令。
>
> **⚠️ 改动它，会发生什么**  
> - 写得太笼统（"帮我写个网站"）→ AI 每轮都在重做同一件事，没有进展。  
> - 不写停止条件 → 任务完成了循环还在跑，继续烧钱。  
> - 写成多任务清单（1. A  2. B  3. C）→ AI 会反复横跳，建议拆成多个 PROMPT 文件。
>
> **🧪 最小可跑示例**
> ```markdown
> # PROMPT.md
> 目标：在 ./output 下生成一个 TODO CLI 工具，支持 add / list / done。
> 
> 每轮你需要：
> 1. 读 ./progress.md，看上轮做到哪了。
> 2. 推进一小步，并在 ./progress.md 末尾追加本轮做了什么。
> 3. 如果功能已全部完成且 `python output/todo.py --help` 能跑通，
>    在 ./progress.md 首行写 DONE 并退出。
> ```
